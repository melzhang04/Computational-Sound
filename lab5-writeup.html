Generative Music with Cellular Automata

For this lab, I implemented a 2D cellular automata system and mapped its output to sound using the Web Audio API. The system generated a grid of cells that evolved over time, and each generation was interpreted as a sequence of notes.

Cellular Automata Implementation

The grid was 12 rows by 16 columns. Each row represented a pitch, and each column represented a time step. I implemented a generalized version of a cellular automaton where the user could control three parameters: the birth condition, and the minimum and maximum survival values.

For each cell, I counted its 8 neighbors with wrap-around at the edges. A dead cell became alive if it had exactly the birth number of neighbors. A live cell survived if its neighbor count was within the survival range. Otherwise, it became dead.

Mapping to Music

Each column was played as a group of notes. Active cells were mapped to frequencies using a selected scale and root note. The scale was defined as semitone offsets and converted to frequency using the standard formula.

To improve sound quality, I limited playback to at most two notes per column and filtered for notes that were spaced apart in pitch. This avoided dense and dissonant clusters.

Playback System

I used the Web Audio API to schedule playback. Each note used an oscillator, gain envelope, and low-pass filter. I applied a short attack and exponential decay to make notes smoother.

I also added a convolution reverb and a compressor. The reverb was generated procedurally, and both reverb and volume were adjustable through the interface.

Improvements to Sound

  • Reduced initial grid density
  • Limited polyphony to two notes
  • Shortened note duration
  • Lowered volume and reverb

These changes made the output more controlled and less chaotic.

Handling Repetition

Cellular automata often became repetitive. I tracked recent note patterns and detected when the system was repeating. When that happened, I modified a small section of the grid to reintroduce variation. When this doesn't work, I also included a randomize button to reintroduce variation as well.

Reflection

The automaton itself only generated structure. The musical result depended heavily on how that structure was mapped to sound. Limiting complexity and adding constraints had the biggest impact on making the output more musical.
